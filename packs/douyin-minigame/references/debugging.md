# 常见问题排查指南

## 开发环境问题

### 开发者工具无法登录

| 现象 | 排查方向 |
|------|---------|
| 扫码后无反应 | 检查抖音 App 版本是否最新 |
| 提示账号未注册 | 确认已在开放平台完成注册 |
| 登录超时 | 检查网络连接，尝试切换登录方式（手机号/邮箱）|

### 项目创建失败

- **AppID 无效**：确认 AppID 来自「开发管理 → 开发设置」，不要用错成小程序的 AppID
- **工程目录问题**：路径不要包含中文或特殊字符

### 预览/编译报错

- **模块找不到**：检查 `game.json` 中的路径配置是否正确
- **Canvas 为空**：确保使用 `tt.createCanvas()` 而非 `document.createElement('canvas')`
- **API 不存在**：检查基础库版本是否满足 API 最低要求

## 支付相关问题

### tt.requestGamePayment 调用失败

| 错误码 | 含义 | 解决 |
|--------|------|------|
| 1 | 系统错误 | 重试，如持续出现联系平台 |
| 2 | 参数错误 | 检查 mode/buyQuantity 等参数格式 |
| 3 | 用户取消支付 | 正常流程，不需处理 |
| 4 | 网络错误 | 检查网络连接 |
| 9 | 订单已支付 | 检查是否重复发起支付 |

### 支付回调收不到

1. 检查服务端回调地址是否正确配置
2. 确认服务端使用 HTTPS
3. 确认回调接口响应格式正确（返回 `{"err_no": 0}`）
4. 检查签名验证逻辑是否正确

> **重要**：支付回调必须在服务端验证签名，客户端验证会被审核拒绝。

## 广告相关问题

### 激励视频不展示

1. **创建时机**：必须在游戏初始化完成后创建广告组件（不要在全局作用域顶层同步创建），否则会静默失败
2. **广告位 ID**：确认广告位 ID 正确且已审核通过
3. **填充率**：新广告位可能需要一段时间才有填充，测试时可能无广告返回
4. **频率限制**：同一用户短时间内多次请求可能被限流

```javascript
// 正确做法：在游戏初始化完成后创建（如引擎的 onLoad 回调中）
const rewardedAd = tt.createRewardedVideoAd({
  adUnitId: '你的广告位ID'
});

rewardedAd.onLoad(() => {
  console.log('广告加载成功');
});

rewardedAd.onError((err) => {
  console.error('广告加载失败:', err);
});

// 展示广告
rewardedAd.show().catch(() => {
  // 广告未加载完成，先加载再展示
  rewardedAd.load().then(() => rewardedAd.show());
});
```

### Banner 广告不显示

- 检查广告位 ID 是否正确
- 检查 Banner 尺寸和位置设置是否在可见区域内
- 检查是否有其他 UI 元素遮挡

## 社交功能问题

### tt.getUserProfile 获取失败

- **用户拒绝授权**：需要在用户点击事件中调用，不能在页面加载时自动调用
- **API 不存在**：确认基础库版本 >= 2.30.0
- 注意：`tt.getUserInfo` 已废弃，必须使用 `tt.getUserProfile`

### 分享功能不生效

- 确认调用了 `tt.shareAppMessage` 或 `tt.showShareMenu`
- 分享图片大小不超过 128KB
- 分享标题不超过 36 个字符

## 审核相关问题

### 审核不通过的常见原因

1. **「必接能力未接入」**
   - 检查侧边栏复访是否正确接入
   - 如果是 IAP 游戏，检查客服能力
   - 如果有用户输入，检查敏感词过滤

2. **「域名不合规」**
   - 所有域名必须 `https://` 或 `wss://`
   - 域名需要在开发者后台的「开发设置 → 服务器域名」中配置

3. **「包体过大」**
   - 包体限制 4MB
   - 解决：使用分包加载，将非核心资源放到 CDN

4. **「功能不完整」**
   - 确保所有页面可正常访问
   - 确保支付流程完整（下单→支付→发货→确认）
   - 确保广告展示正常

## 性能问题

### 加载缓慢

| 优化方向 | 具体做法 |
|---------|---------|
| 图片资源 | PNG 格式 + 控制尺寸 + 裁剪透明区域 |
| 纹理压缩 | 使用 ETC2（手机端）/ S3TC / PVRTC |
| 分包加载 | 核心代码 ≤ 4MB，其余按需加载 |
| 合并小图 | 小图合并为图集（≤ 1024x1024，利用率 > 75%）|

### 运行卡顿

- **内存泄漏**：检查纹理和音频资源是否及时释放
- **绘制过多**：减少每帧 draw call 数量
- **主线程阻塞**：将计算密集任务移到 Worker 线程

### 闪退

- 通常是内存溢出导致
- 纹理大小计算：`长 × 宽 × 通道数 × 通道位数`
- 解决：使用压缩纹理、按需加载/释放资源、减小图片尺寸
