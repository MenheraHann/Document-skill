# 提审上线全流程

## 提审前必做检查清单

在提交审核前，逐项确认：

| 检查项 | 说明 | 必须 |
|--------|------|------|
| 侧边栏复访 | 已接入侧边栏能力 | 所有游戏 |
| 客服能力 | 已接入客服入口 | IAP 游戏 |
| 敏感词过滤 | 对用户输入做了过滤 | 有用户输入的游戏 |
| 域名 HTTPS | 所有域名用 `https://` 或 `wss://` | 所有游戏 |
| 基础信息 | 名称/头像/简介/软著全部审核通过 | 所有游戏 |
| 隐私政策 | 已设置隐私政策 | 所有游戏 |
| 适龄提示 | 已设置适龄提示 | 所有游戏 |
| 自审自查 | 已完成自审自查确认 | 所有游戏 |
| 包体大小 | 不超过 4MB（超出用分包加载）| 所有游戏 |
| 测试代码 | 移除 console.log 和测试代码 | 所有游戏 |
| 防沉迷 | 接入中宣部防沉迷实名认证系统 | 所有游戏 |

## 版本阶段说明

| 版本阶段 | 说明 |
|---------|------|
| 测试版本 | 开发完成上传到平台，扫码测试体验 |
| 审核版本 | 测试版本提交审核，等待平台审核 |
| 灰度版本 | 面向部分用户的线上版本（需先有线上版本）|
| 线上版本 | 面向全部用户的正式版本 |

## 提审流程

### 1. 上传游戏包

**普通小游戏**：在开发者工具中点击「上传」按钮，填写版本号和描述。

**Unity 小游戏**：通过 Unity 小游戏转换工具导出后，在开发者工具中上传。

> 最多支持 25 个测试版本并行，支持多通道测试。

### 2. 测试验证

上传后在开发者后台生成测试二维码，用抖音扫码测试：
- 验证核心玩法流程
- 验证支付/广告功能
- 验证在不同设备上的表现

### 3. 提交审核

1. 登录 [开发者控制台](https://developer.open-douyin.com/console)
2. 进入小游戏 → 版本管理
3. 选择已测试完成的版本 → 提交审核
4. 填写审核说明（描述游戏玩法、变现方式等）

### 4. 审核结果处理

| 结果 | 处理 |
|------|------|
| 审核通过 | 可选择全量发布或灰度发布 |
| 审核不通过 | 根据拒绝原因修改后重新提交 |

### 常见审核拒绝原因

1. **未接入必接能力** — 侧边栏/客服/敏感词未接入
2. **域名不合规** — 使用了 http 而非 https
3. **缺少软著** — 未提供软件著作权登记证书
4. **内容违规** — 游戏内容不符合平台规范
5. **功能不完整** — 游戏崩溃、核心功能无法使用
6. **包体过大** — 超过 4MB 未做分包

### 5. 发布上线

**全量发布**：所有用户可见，适合功能稳定的版本。

**灰度发布**：面向部分用户，适合大更新的渐进式发布。前提是已有线上版本。

## 发布后数据监控

发布后可在开发者后台查看：

| 数据类型 | 包含指标 |
|---------|---------|
| 总览数据 | DAU、新增用户、留存率 |
| 用户分析 | 用户画像、行为路径 |
| 性能分析 | 启动耗时、帧率、崩溃率 |

## 版本更新流程

1. 开发新版本 → 上传测试版本
2. 测试通过 → 提交审核
3. 审核通过 → 全量/灰度发布
4. （可选）配置版本更新提醒，通知用户更新

## 上线后运营推广

| 方式 | API / 操作 | 说明 |
|------|-----------|------|
| 搜索配置 | 开发者后台设置标签 | 优化搜索排名 |
| 直玩获客 | 平台自动推荐 | 推荐流中直接试玩 |
| 订阅消息 | `tt.requestSubscribeMessage` | 向用户发送游戏通知 |
| 关注抖音号 | `tt.openAwemeUserProfile` | 引导关注官方号 |
| 粉丝群聊 | 开发者后台配置 | 建立玩家社区 |
